Teknisk Översikt

I detta projekt har jag använt följande tekniker:

  • HTML5 Canvas: Canvas används för att rita och ändra 2D-grafik. Jag har använt Canvas för att skapa spelets visuella delar, som bollen, paddeln och brickorna. Canvas API låter mig uppdatera och animera dessa delar för att göra spelet interaktivt.
  • CSS: Min CSS är designad för att vara responsiv och användarvänlig på olika enheter. Jag använder Flexbox för att centrera innehållet både horisontellt och vertikalt, vilket gör att layouten anpassar sig till skärmstorlek. Bakgrundsgradienter och transparenta element skapar en modern design. Knappar och text har anpassade storlekar och marginaler för bättre användarupplevelse, och modala fönster är implementerade för att hantera olika interaktioner smidigt. Allt detta säkerställer att webbplatsen ser bra ut och fungerar väl på både stora skärmar och mobila enheter.
  • JavaScript: JavaScript är språket som styr spelets logik och interaktivitet. Jag har använt JavaScript för att hantera användarens tangenttryckningar, kollisionsdetektion och spelets flöde. JavaScript gör att spelet reagerar på användarens input och uppdaterar spelets tillstånd i realtid.
  • Tutorial och Extern Kodbas: Projektet började med inspiration från en tutorial och en befintlig kodbas. Genom att studera dessa resurser kunde jag ställa frågor och hitta lösningar på specifika problem, vilket hjälpte mig att bygga mitt spel.

Genom att kombinera dessa tekniker har jag skapat ett spel som är både interaktivt och visuellt tilltalande. Detta projekt har också gett mig en bättre förståelse för hur dessa verktyg kan användas tillsammans för att skapa dynamiska webbapplikationer.

Canvas

I detta projekt har jag använt HTML5 Canvas för att skapa och hantera 2D-grafik på webbplatsen. Canvas är en kraftfull teknik som gör det möjligt att rita och animera grafik direkt i webbläsaren. Jag har använt Canvas för att utveckla spelet och skapa interaktiva element.
Med Canvas kan man rita former, bilder och text, samt skapa animationer och spel. I mitt projekt använder jag Canvas för att rita spelplanen, bollen, paddeln och tegelstenarna. Genom att kombinera olika Canvas-funktioner kan jag skapa ett dynamiskt och interaktivt spel.
Detta är första gången jag arbetar med Canvas, och det har varit en lärorik upplevelse. Jag har lärt mig mycket om hur man använder Canvas för att skapa grafik och hur man programmerar spel och animationer. Det har varit spännande att se hur man kan skapa interaktiva element med bara några rader kod:

  • Bollen: Ritas som en cirkel som rör sig enligt spelets regler.
  • Paddeln: Ritas som en rektangel som användaren styr med tangenterna.
  • Brickorna: Ritas som rektanglar som bryts när bollen träffar dem.
  • Stjärnorna: Ritas på en annan Canvas för att skapa en rörlig bakgrund.

Med Canvas API kan jag enkelt rita, uppdatera och animera dessa objekt, vilket gör spelet mer dynamiskt och interaktivt.

Agile-metoden

För att utveckla projektet effektivt använde jag Agile-metoden. Den gör att jag kan arbeta med olika delar av koden steg för steg och snabbt anpassa mig efter förändringar och feedback. Genom att jobba i korta omgångar och ständigt testa och förbättra koden kunde jag skapa och förbättra spelet på ett effektivt sätt.

Användning av ChatGPT som Verktyg

I det här projektet har jag använt ChatGPT för att få hjälp med vissa utmaningar och för att få feedback på mina idéer och lösningar. ChatGPT har varit till nytta för att förklara svåra koncept, ge förslag på hur jag kan strukturera koden och ge tips om hur jag kan förbättra delar av spelet.

Jag har dock använt dessa tips som en del av min egen arbetsprocess. Jag har anpassat och implementerat förslagen så att de passar just mitt projekt.

Det är viktigt att förstå att ChatGPT har fungerat som en rådgivare och inte som den huvudsakliga skaparen av projektet. Jag har själv utvecklat de flesta av lösningarna och gjort egna beslut om hur jag ska använda verktyget. Jag ser det som ett stöd som hjälper mig i den kreativa processen, snarare än som en ersättning för min egen insats.
